## 10. Type System Design Discussion

### User
> i think we can add some other type too: example target to buy | invest. Is these a good idea and are they consider as a type? And income, expense will be a type too?

### Assistant
Explained using Sebesta's terminology:

- **`income`/`expense`** are NOT types — they are values of an **enumeration type** (`transaction_kind = { income, expense }`)
- **`target`** (savings goal) CAN be a second **structured type** (record with name, goal, saved, deadline)
- **`invest`** works better as a **domain-specific command** than a type

Suggested final type system:

| Category | Types |
|----------|-------|
| Primitive | `money`, `percent`, `int`, `string`, `bool` (5 types) |
| Enumeration | `transaction_kind = { income, expense }` |
| Structured | `transaction` (record), `target` (record) |
| Domain constructs | `budget`, `track`, `report`, `alert`, `invest` (commands) |

User accepted as starting point with flexibility to change later.
